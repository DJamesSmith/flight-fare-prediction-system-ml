Project Summary

The Flight Fare Prediction System is a machine learning application designed to estimate airline ticket prices based
on historical flight information. The system analyzes several factors such as airline company, source city, destination
city, journey date, departure time, arrival time, number of stops, and flight duration to predict the expected ticket fare.

The project demonstrates how machine learning can assist travelers in estimating airfare before booking tickets,
enabling informed travel planning and cost optimization. The application highlights the complete machine learning workflow,
including data preprocessing, feature encoding, model training, prediction, and evaluation.

The project follows Object-Oriented Programming concepts, modular architecture, separation of concerns, and
proper validation techniques to ensure maintainability and scalability.

1. the admin manages datasets, model training, evaluation, reports, and users;
2. the user only performs predictions and views their own history;
3. every major stage of the ML pipeline is encapsulated in its own service;
4. reports, trained models, and prediction history are persisted to files (and optionally a database).

----------------------------------------------------------------------------------------------------

Business Problem

Airline ticket prices fluctuate continuously due to demand, seasonality, holidays, airline competition, and booking time.
This project helps users estimate ticket prices before booking, allowing them to compare options and make informed travel decisions.

----------------------------------------

2. Data Cleaning
Real-world datasets are rarely perfect. Data cleaning prepares the dataset before training.

Before cleaning
| Airline | Duration | Fare |
| ------- | -------- | ---- |
| Indigo  | 2h       | 5200 |
| NULL    | 3h       | 6100 |
| Indigo  | 2h       | 5200 |

Problems:
1. Missing airline name
2. Duplicate record
3. Invalid values
4. Extra spaces
5. Incorrect date formats

After cleaning:
| Airline | Duration | Fare |
| ------- | -------- | ---- |
| Indigo  | 2h       | 5200 |

----------------------------------------
3. Train ML Model

Input validation:
Clean data
Features
Correct flight fares

The algorithm studies the relationship between them.
For example, it may learn:

1. Longer flights usually cost more.
2. Non-stop flights are often more expensive.
3. Flights during holidays have higher fares.
4. Certain airlines tend to charge more.

Sole purpose of this training process is to discover patterns that can later be used to predict fares for new flights.

----------------------------------------
4. Save trained_model.pkl

Once the model has learned, it is saved to disk.
Example:
trained_model.pkl

A .pkl (pickle) file stores the trained model so it can be reused later.
Without saving it, the model would need to retrain every time the application starts, which is inefficient.

This is to preserve the trained model for future predictions.

End of Admin Training Model.
----------------------------------------

5. User Enters a NEW Flight
The user provides details of a flight that was not part of the training dataset.
Input of Airline, Indigo, Source, Kolkata, Destination, Delhi, Departure, 09:30, Arrival, 11:45, Stops, 0

----------------------------------------

6. Load trained_model.pkl
Instead of training again, the application simply loads the saved model from the .pkl file.

Conceptually:
Open trained_model.pkl
↓
Restore trained model into memory
↓
Ready to predict

This process is very fast and avoids unnecessary retraining.
Helps to reuse the previously trained model for making predictions.

----------------------------------------
8. Predict Flight Fare
The user's flight details are passed to the loaded model.
The model compares the new flight with the patterns it learned during training.

Example reasoning (simplified):
This looks similar to previous Indigo flights...
Non-stop...
2-hour duration...
October travel...
Expected fare...
₹5,480

This prediction is generated by the algorithm based on learned relationships in the data.
